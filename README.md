# 리눅스 명령어 정리
## top
- 시스템의 상태를 전반적으로 가장 빠르게 파악 가능
    - CPU, Memory, Process

- 옵션 없이 입력하면 interval 간격(기본 3초)으로 화면을 갱신하여 정보를 보여줌
### 실행 전 옵션
    - 순간의 정보 확인 : -b (batch 모드)
    - -n : top 실행 주기 설정

### 실행 후 명령어
    - shift + p : CPU 사용률 내림차순
    - shift + m : 메모리 사용률 내림차순
    - shift + t : 프로세스가 돌고있는 시간 순
    - k : kill(후술함) k 입력 후 PID 번호 작성
    - f : sort field 선택 화면
        - q 누르면 RES 순으로 정렬
            - RES : 프로세스가 물리적 메모리를 사용하는 크기
    - a : 메모리 사용량 순으로 정렬
    - b : Batch 모드로 작동
    - 1 : CPU Core별 사용량

## ps
- Process Status
- 현재 돌아가고 있는 프로세스 확인
    - PID, TTY(프로세스가 연결된 터미널), 총 CPU 사용 시간, 실행 명령행

### 옵션
    - -A        : 모든 프로세스 출력
    - a         : 터미널과 관련된 프로세스 출력
    - -a        : 로그인 셸을 제외하고 데몬 등 터미널에 종속되지 않은 모든 프로세스 출력
    - -e        : 커널을 제외한 모든 프로세스 출력
    - -f        : 유닉스 스타일로 출력 (UID, PID, PPID 등이 함께 표시)
    - -l / l    : 프로세스의 정보를 길게 보여줌. 우선순위와 관련된 값 확인 가능
    - -o value  : 출력 포맷 지정
    - -M        : 64비트 프로세스 표시
    - -m        : 커널 스레드도 표시
    - -p        : 특정 PID 지정
    - -r        : 현재 실행 중인 프로세서 표시
    - u         : 프로세스의 소유자를 기준으로 출력
    - -u        : 특정 사용자의 프로세스 정보를 확인
    - x         : 데몬처럼 터미널에 종속되지 않는 프로세스 출력
    - -x        : 로그인 하고있는 동안 아직 완료되지 않은프로세서 표시

### 옵션
    - USER      : 프로세스 소유자 이름 (BSD 계열)
    - UID       : 프로세스 소유자 이름 (SYSTEM V 계열)
    - PID       : 프로세스 식별 번호
    - PPID      : 부모 프로세스 ID
    - %CPU      : CPU 사용 비율의 추정치
    - %MEM      : 메모리 사용 비율 추정치
    - VSZ       : K / 페이지 단위의 가상메모리 사용량
    - RSS       : 실제 메모리 사용량
    - TTY       : 프로세스와 연결된 터미널
    - S / STAT  : 현재 프로세스의 상태 코드
    - TIME      : 총 CPU 사용 시간
    - COMMAND   : 프로세스의 실행 명령행
    - STIME     : 프로세스가 시작된 시간 / 날짜
    - C / CP    : 짧은 기간 동안의 CPU 사용륭
    - F         : 프로세스의 플래그
    - PRI       : 실제 실행 우선순위
    - NI        : nice 우선순위 번호

## jobs
- 작업의 상태를 표시

### 상태
    - Running           : 작업이 계속 진행중임
    - Done              : 작업이 완료되어 0을 반환
    - Done(code)        : 작업이 종료되었으며 0이 아닌 코드를 반환
    - Stopped           : 작업이 일시 중단
    - Stopped(SIGTSTP)  : SIGTSTP 시그널이 작업을 일시 중단
    - Stopped(SIGSTOP)  : SIGSTOP 시그널이 작업을 일시 중단
    - Stopped(SIGTTIN)  : SIGTTIN 시그널이 작업을 일시 중단
    - Stopped(SIGTTOU)  : SIGTTOU 시그널이 작업을 일시 중단

### 옵션
    - -l        : 프로세스 그룹 ID를 state 필드 앞에 출력
    - -n        : 프로세스 그룹 중에 대표 프로세스 ID를 출력
    - -p        : 각 프로세스 ID에 대해 한 행씩 출력
    - command   : 지정한 명령어를 실행
    
## kill
- 프로세스를 지정하고 신호를 보내 제어
- 주로 프로세스를 종료하는 용도로 사용

### 옵션
    - -9 [PID]  : 
    - -l        : 

### 대표적 시그널 종류
| 번호 | 신호 이름 | 의미 |
| ---- | -------- | ------- |
| 1 | SIGHUP | 프로그램이 사용하는 설정 파일 변경 |
| 2 | SIGINT | 프로그램의 동작 멈춤 |
| 9 | SIGKILL | 프로그램을 무조건 종료 |
| 11 | SIGSEGV | 잘못된 메모리 관리시 생기는 신호 |
| 15 | SIGTERM | 실행중인 프로그램을 정상적인 종료 방법으로 프로그램을 종료 |
| 18 | SIGCONT | 중지 되어 있는 프로그램을 재실행 |
| 19 | SIGSTOP | 프로그램을 중지 |
| 20 | SIGTSTP | 터미널에서 중지 |
